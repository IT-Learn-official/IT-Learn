# Version Control, Git, and GitHub

When you code, you change files all the time.
Version control keeps a history, so you can go back when something breaks.

**Git** is the tool on your computer.
It tracks changes in a folder called a **repository** (repo).

**GitHub** is a website that can store your Git repo online.
It helps you back up your work and work with other people.

---

The big idea in Git is a **commit**.
A commit is a saved snapshot with a short message.

Terms to know:

```text
Files you edit  -> working directory
Saved snapshots -> commits
Online copy     -> remote (GitHub)
```
