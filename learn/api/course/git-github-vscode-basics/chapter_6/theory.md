# Collaboration: Pull Requests, Reviews, and Issues

On a team, you usually do not push changes straight into `main`.
You use a branch and a pull request.

**Pull request (PR)**: "Please merge my branch into main."
It shows what changed, and lets people comment.

**Review**: teammates can approve or ask for changes.

**Issues**: a shared list for bugs and ideas.

---

If you do not have permission to push, you can **fork** the repo.
That makes your own copy on GitHub, so you can open a PR back to the original.
