# Kapitel 1: Das Web vs. Das Internet (und wer mit wem spricht)

Dieses Kapitel erklärt den Unterschied zwischen dem Internet und dem Web und führt das Client/Server‑Anfrage/Antwort‑Muster ein.

Am Ende dieses Kapitels wirst du in der Lage sein:

- Den Unterschied zwischen dem **Internet** und dem **World Wide Web** zu erklären.
- Das **Client/Server**‑Prinzip (Browser fragt, Server antwortet) zu beschreiben.
- Zusammenzufassen, was eine **Anfrage** und eine **Antwort** sind.

---

## Internet vs. Web (kurz)

Das **Internet** ist das globale Netzwerk verbundener Geräte und Kabel. Das **Web** ist ein Dienst, der auf diesem Netzwerk läuft: Webseiten, Links und Seiten, die Browser über HTTP laden.

Kurz: Internet = Netzwerk; Web = Webseiten und Web‑Apps.

> **Wichtig:** Das Web nutzt das Internet, aber das Internet ist größer als das Web.

---

## Client und Server

- **Client:** ein Browser oder eine App, die nach Daten fragt.
- **Server:** eine Maschine oder ein Dienst, der Anfragen beantwortet.

Analogie: Du (Client) bestellst eine Pizza; das Restaurant (Server) bereitet sie zu und liefert.

---

## Anfrage und Antwort

Typischer Ablauf:

1. Der Client sendet eine **Anfrage** (z. B. `GET /index.html`).
2. Der Server liefert eine **Antwort** (Statuscode, Header und Body).

Spätere Kapitel erklären die Teile von Anfragen und Antworten genauer.

> **Wichtig:** Webseiten und APIs funktionieren, indem Clients Anfragen stellen und Server Antworten senden.
