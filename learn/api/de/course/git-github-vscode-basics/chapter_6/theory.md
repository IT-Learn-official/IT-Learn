# Zusammenarbeit: Pull Requests, Reviews und Issues

Im Team pushst du Änderungen meistens nicht direkt in `main`.
Du nutzt einen Branch und einen Pull Request.

**Pull Request (PR)**: „Bitte merge meinen Branch in main.“
Er zeigt, was sich geändert hat, und ermöglicht Kommentare.

**Review**: Teammitglieder können freigeben oder Änderungen verlangen.

**Issues**: eine gemeinsame Liste für Bugs und Ideen.

---

Wenn du keine Berechtigung zum Pushen hast, kannst du das Repo **forken**.
Damit bekommst du deine eigene Kopie auf GitHub und kannst eine PR zurück zum Original öffnen.
