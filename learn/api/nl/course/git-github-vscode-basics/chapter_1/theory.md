# Versiebeheer, Git en GitHub

Als je code schrijft, pas je voortdurend bestanden aan.
Versiebeheer houdt een geschiedenis bij, zodat je kunt teruggaan wanneer er iets misloopt.

**Git** is het hulpmiddel op je computer.
Het volgt wijzigingen in een map die een **repository** (repo) heet.

**GitHub** is een website die je Git-repo online kan bewaren.
Zo heb je een back-up en kan je samen met anderen aan hetzelfde project werken.

---

Het grote idee in Git is een **commit**.
Een commit is een opgeslagen momentopname met een korte boodschap.

Begrippen om te kennen:

```text
Bestanden die je wijzigt  -> werkmap
Opgeslagen momentopnames  -> commits
Online kopie              -> remote (GitHub)
```
