# Samenwerken: pull requests, reviews en issues

In een team push je meestal niet rechtstreeks naar `main`.
Je gebruikt een branch en een pull request.

**Pull request (PR)**: "Gelieve mijn branch te mergen in main."
Het toont wat er veranderde en laat mensen reageren.

**Review**: teamgenoten kunnen goedkeuren of wijzigingen vragen.

**Issues**: een gedeelde lijst voor bugs en ideeën.

---

Als je geen rechten hebt om te pushen, kan je de repo **forken**.
Dan maak je je eigen kopie op GitHub en kan je een PR terug openen naar het origineel.
